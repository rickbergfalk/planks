Tooltip

A popup that displays information related to an element when the element receives keyboard focus or the mouse hovers over it.

Basic Usage

Hover me Add to library 
<plank-tooltip>
  <plank-tooltip-trigger>
    <plank-button variant="outline">Hover me</plank-button>
  </plank-tooltip-trigger>
  <plank-tooltip-content>Add to library</plank-tooltip-content>
</plank-tooltip>

Different Sides

Top Top tooltip Right Right tooltip Bottom Bottom tooltip Left Left tooltip 
<plank-tooltip-content side="top">...</plank-tooltip-content>
<plank-tooltip-content side="right">...</plank-tooltip-content>
<plank-tooltip-content side="bottom">...</plank-tooltip-content>
<plank-tooltip-content side="left">...</plank-tooltip-content>

With Delay

Use the delay-duration attribute to add a delay before the tooltip appears.

500ms delay This has a 500ms delay 
<plank-tooltip delay-duration="500">
  ...
</plank-tooltip>

Controlled State

Use the open attribute to control the tooltip state programmatically.

Always open This tooltip is controlled 
<plank-tooltip open>
  ...
</plank-tooltip>

API

plank-tooltip

Property Type Default Description
open boolean false The controlled open state
delay-duration number 0 Delay in ms before showing the tooltip

Events

Event Detail Description
open-change { open: boolean } Fired when the open state changes

plank-tooltip-trigger

The element that triggers the tooltip. Shows tooltip on hover and focus.

plank-tooltip-content

Property Type Default Description
side "top" | "right" | "bottom" | "left" "top" The preferred side for the tooltip
side-offset number 0 Distance from the trigger in pixels

Accessibility

  • Tooltip content has role="tooltip"
  • Trigger element has aria-describedby pointing to the tooltip when open
  • Shows on both hover and keyboard focus
  • Uses Floating UI for smart positioning that avoids viewport edges
