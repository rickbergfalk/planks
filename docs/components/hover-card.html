With Custom Delays

Use the open-delay and close-delay attributes to customize timing. Default is 700ms open delay and 300ms close delay.

No delay

This appears instantly!

 1 second delay

This took a full second to appear.

 
<hal-hover-card open-delay="0" close-delay="0">
  ...
</hal-hover-card>

<hal-hover-card open-delay="1000">
  ...
</hal-hover-card>

Controlled State

Use the open attribute to control the hover card state programmatically.

Always open

This hover card is always open.

 
<hal-hover-card open>
  ...
</hal-hover-card>

API

hal-hover-card

Property Type Default Description
open boolean false The controlled open state
open-delay number 700 Delay in ms before showing the hover card
close-delay number 300 Delay in ms before hiding the hover card

Events

Event Detail Description
open-change { open: boolean } Fired when the open state changes

hal-hover-card-trigger

The element that triggers the hover card. Shows on hover (mouse only) and keyboard focus.

hal-hover-card-content

Property Type Default Description
side "top" | "right" | "bottom" | "left" "bottom" The preferred side for the hover card
align "start" | "center" | "end" "center" The alignment of the hover card
side-offset number 4 Distance from the trigger in pixels

Accessibility

  • Opens on keyboard focus for keyboard users
  • Does not open on touch to avoid interfering with taps
  • Trigger has data-state attribute indicating open/closed state
  • Uses Floating UI for smart positioning that avoids viewport edges
  • Stays open when hovering over the content for easy interaction
