Dialog

A modal dialog that interrupts the user with important content and expects a response.

Basic Usage

Edit Profile Edit profile Make changes to your profile here. Click save when you're done.

Dialog content goes here.

Cancel Save changes 
<plank-dialog>
  <plank-dialog-trigger>
    <plank-button variant="outline">Edit Profile</plank-button>
  </plank-dialog-trigger>
  <plank-dialog-content>
    <plank-dialog-header>
      <plank-dialog-title>Edit profile</plank-dialog-title>
      <plank-dialog-description>
        Make changes to your profile here. Click save when you're done.
      </plank-dialog-description>
    </plank-dialog-header>
    <div class="py-4">
      <p class="text-sm">Dialog content goes here.</p>
    </div>
    <plank-dialog-footer>
      <plank-button variant="outline">Cancel</plank-button>
      <plank-button>Save changes</plank-button>
    </plank-dialog-footer>
  </plank-dialog-content>
</plank-dialog>

Without Close Button

Use show-close-button="false" to hide the default close button.

Open Dialog No Close Button This dialog has no default close button. Press Escape or click outside to close. 
<plank-dialog-content show-close-button="false">
  ...
</plank-dialog-content>

Custom Styling

Use the class attribute to add custom styles.

Wide Dialog Wide Dialog This dialog uses a custom max-width.

Content can be wider with sm:max-w-2xl or other utilities. 

<plank-dialog-content class="sm:max-w-2xl">
  ...
</plank-dialog-content>

API

plank-dialog

Property Type Default Description
open boolean false The controlled open state

Events

Event Detail Description
open-change { open: boolean } Fired when the open state changes

plank-dialog-trigger

The element that triggers the dialog. Opens dialog on click.

plank-dialog-content

Property Type Default Description
show-close-button boolean true Whether to show the close button
class string "" Additional CSS classes

plank-dialog-header

Container for title and description. Provides proper spacing.

plank-dialog-footer

Container for action buttons. Stacks on mobile, row on desktop.

plank-dialog-title

The dialog title. Used for aria-labelledby.

plank-dialog-description

The dialog description. Used for aria-describedby.

Accessibility

  • Dialog content has role="dialog"
  • Trigger has aria-haspopup="dialog"
  • Trigger has aria-expanded to indicate open state
  • Trigger has aria-controls pointing to the dialog when open
  • Dialog has aria-labelledby pointing to the title
  • Dialog has aria-describedby pointing to the description
  • Closes on Escape key
  • Closes when clicking on overlay
  • Modal blocks interaction with content behind
