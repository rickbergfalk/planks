# On AI Use in This Project

This project was developed with significant AI assistance. Claude specifically. I want to be transparent about that, and about my conflicted feelings on the subject.

I don't want this project signaling I approve of the industry behind this technology, because I do not. At the same time, I also want to acknowledge that the technology is actually useful, and genuinely allowed me(?) to build this. **_Me_**... I'm even disgusted by that phrasing because **I** didn't actually write any code. Just documents and instruction after reading responses and output.

The only reason this all was possible was because all of our open source code and other text was vacuumed up, processed with a lot of electricity, and sold back to us.

## What AI was used for

Basically everything:

- Cloning, cleanup, and restructuring of the shadcn project
- Documenting migration strategy
- Standing up the testing frameworks
- Writing tests of shadcn react components
- Rapid prototyping of web component implementations
- Debugging and iterating on test failures
- Exploring Lit patterns and light DOM constraints
- General "pair programming" on repetitive conversion work

It is easier to acknowledge what **I** did:

- Wrote prompts detailing approaches I wanted to take
- Thought about what I wanted to be done, then asked Claude pointed questions
- Provided references with things I've read and thought through
- Reviewed the work... though let's be honest here. So far I've really only reviewed the tangible outputs. Often my review is asking follow up questions I know the answer to, and making sure Claude and I are on the same page.

## My concerns

I **really** don't like the state of LLM technology:

- **Energy consumption** - Training and running these models uses substantial power
- **Training data** - Models are trained on work created by others, often without permission or compensation
- **Labor displacement** - These tools may contribute to job losses in knowledge work
- **Industry dynamics** - Using AI tools supports and legitimizes an industry I'm ambivalent about
- **Loss of creativity** - The output is recycled content of what already exists, stealing an artist's creation
- **Dangerous uses** - AI psychosis, delusions, reinforcement of false information.
- **Hype cycles** - I don't enjoy contributing to tech hype

## Why I used it anyway

The goal of reducing dependence on React felt worth pursuing. Claude made this possible to do in days/weeks instead of months/years. I may not have even bothered to explore this idea.

Claude says using a tool while questioning the industry behind it isn't hypocrisy but rather living with contradictions.

I think it helps that this project is code. Open source code licenses allow for code to be used a variety of ways. This project isn't original art. It's setting up an assembly line and watching the results. Figuring out how to apply the techniques to guide the work was the creativity.

I want to be open and honest about all this, because I really hope that someday we could have a good ethical LLM that is used responsibly. Or at the very least, nuance on the internet.
